# web-site-resource

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面不针对任何特定域名或网站，而是作为通用资源存放地，方便在不同场景下快速访问和分发。

仓库中的每个 HTML 文件均为独立、自包含的页面，无需额外依赖即可在浏览器中直接打开运行。适合作为静态页面存档、临时演示、工具页面或信息展示用途。

## 目录说明

```
web-site-resource/
├── pages/              # 存放所有归档的 HTML 页面文件
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/             # 页面引用的公共资源（图片、样式、脚本等）
│   ├── css/
│   ├── js/
│   └── images/
├── docs/               # 附加文档或说明（可选）
└── README.md           # 本文件
```

- `pages/`：核心目录，所有归档的 HTML 页面均放置于此。
- `assets/`：存放多个页面可能共用的静态资源，便于统一维护。
- `docs/`：可选的文档目录，用于存放与页面相关的说明或参考资料。

## 页面归档说明

- 每个 HTML 页面应保持独立，避免依赖外部不可控资源（如第三方 CDN）。如需引用资源，建议优先放置于 `assets/` 目录下。
- 页面命名建议使用英文小写字母、数字和短横线（`-`），避免特殊字符和中文。
- 页面内容应合法合规，不包含恶意代码、侵权内容或违反平台规定的信息。
- 归档的页面不会自动更新，请通过提交 Pull Request 或直接推送的方式新增或修改页面。

## 维护说明

- 本仓库接受来自社区或个人的页面归档请求。提交前请确保页面内容符合上述规范。
- 维护者会定期检查页面可用性，但不对页面内容的准确性和时效性做出保证。
- 如有问题或建议，欢迎通过 Issues 反馈。
- 仓库历史中可能包含多个版本的页面，请以最新提交为准。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 在 `pages/` 目录下找到目标 HTML 文件。
3. 使用任意浏览器直接打开该文件即可查看。

也可通过 GitHub Pages 或其他静态托管服务部署后在线访问（需自行配置）。
